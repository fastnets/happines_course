# Документация проекта

Этот раздел — основной вход в текущие задачи и ограничения проекта.

## Файлы
- `docs/TASK.md` — текущая цель и критерии приемки.
- `docs/CONSTRAINTS.md` — ограничения и правила разработки.

## Текущий фокус
1. Стабильность и качество: тесты, CI, критичные багфиксы.
2. Закрытие бизнес-пробелов: персональные напоминания, поддержка, ачивки/прогресс.
