# Текущая задача

## Цель
Довести проект до устойчивого MVP, который соответствует ключевым бизнес-требованиям:
- стабильный бот с базовой автоматизацией курса,
- контролируемая админка,
- предсказуемая доставка материалов и начислений,
- воспроизводимое качество через тесты и CI.

## Текущий этап
1. Исправить критичные несоответствия в логике.
2. Закрыть минимальный тестовый контур для ключевых сервисов.
3. Подключить CI, чтобы тесты выполнялись автоматически.

## Следующие этапы
1. Реализовать "Мои напоминания" (персональные напоминания пользователя).
2. Реализовать эскалацию поддержки ("Это не помогло" -> тикет администратору -> ответ пользователю).
3. Реализовать ачивки и расширенный прогресс (включая данные анкет для диаграмм).

## Критерии приемки
- Все unit-тесты в `tests/` проходят локально и в CI.
- Любой push/PR в `main` автоматически запускает тесты.
- Критичные баги в бизнес-логике фиксируются отдельными коммитами с тестами.
